# Database Schema Design (Canteen System)

## 1. User & Tenant Management (Module: Core)

This module handles multi-tenancy isolation and user permissions.

### Table: `ClientCompany`
Represents the tenant entity (e.g., Hospital A, Rehab Center B). All operational data is scoped to a specific company ID.

| Field Name | Type | Key | Description |
| :--- | :--- | :--- | :--- |
| `id` | Integer | **PK** | Primary Key |
| `name` | String(100) | | Official name of the client company |
| `code` | String(20) | **Unique** | Unique identifier (e.g., "HOSP01") |
| `created_at` | DateTime | | Timestamp of creation |

### Table: `UserProfile`
Extends the standard Django Auth User model to add tenant association and roles.

| Field Name | Type | Key | Description |
| :--- | :--- | :--- | :--- |
| `id` | Integer | **PK** | Primary Key |
| `user_id` | Integer | **FK (1:1)** | Links to Django `auth_user` table |
| `company_id` | Integer | **FK** | Links to `ClientCompany`. Defines data access scope. |
| `role` | String(2) | | Permission level: 'RW' (Manager) or 'RO' (Viewer) |

## 2. Master Data: Ingredients & Recipes (Module: Core)

This module stores the standardized data used for calculation logic.

### Table: `Unit`
Measurement units.

| Field Name | Type | Key | Description |
| :--- | :--- | :--- | :--- |
| `id` | Integer | **PK** | Primary Key |
| `name` | String(20) | **Unique** | Unit name (e.g., kg, g, case) |

### Table: `RawMaterial`
Represents the base ingredient as purchased from suppliers (Gross Material).

| Field Name | Type | Key | Description |
| :--- | :--- | :--- | :--- |
| `id` | Integer | **PK** | Primary Key |
| `name` | String(100) | **Unique** | Name of the raw ingredient (e.g., "Potato - Raw") |
| `category` | String(10) | | Category (VEG, MEAT, GRAIN, etc.) |
| `default_unit_id` | Integer | **FK** | Links to `Unit`. The standard purchasing unit. |

### Table: `ProcessedMaterial` (Core Logic)
Defines how a raw material is processed and its corresponding yield rate.
*Formula: Gross Weight = Net Weight / Yield Rate*

| Field Name | Type | Key | Description |
| :--- | :--- | :--- | :--- |
| `id` | Integer | **PK** | Primary Key |
| `raw_material_id` | Integer | **FK** | Links to `RawMaterial` |
| `method_name` | String(50) | | Processing method (e.g., "Peeled & Diced") |
| `yield_rate` | Decimal(5,2) | | **The Yield Rate**. 1.00 = 100%, 0.80 = 80% |

**Constraints:**
- `unique_together = ('raw_material_id', 'method_name')` - Ensures distinct processing methods for the same material

### Table: `DietCategory`
Classifies the type of set meals provided.

| Field Name | Type | Key | Description |
| :--- | :--- | :--- | :--- |
| `id` | Integer | **PK** | Primary Key |
| `name` | String(50) | **Unique** | Name (e.g., "Standard Diet", "Diabetic") |

### Table: `Dish`
Represents a specific dish item.

| Field Name | Type | Key | Description |
| :--- | :--- | :--- | :--- |
| `id` | Integer | **PK** | Primary Key |
| `name` | String(100) | **Unique** | Dish name (e.g., "Tomato Beef Stew") |
| `allowed_diets` | M2M | **M2M** | Many-to-Many link to `DietCategory` |

### Table: `DishIngredient` (BOM / Recipe)
The Bill of Materials for a dish. Each ingredient links directly to a raw material with an optional processing method.

| Field Name | Type | Key | Description |
| :--- | :--- | :--- | :--- |
| `id` | Integer | **PK** | Primary Key |
| `dish_id` | Integer | **FK** | Links to `Dish` |
| `raw_material_id` | Integer | **FK** | Links to `RawMaterial` (直接关联原料) |
| `processing_id` | Integer | **FK (nullable)** | Links to `ProcessedMaterial` (可选的处理方法) |
| `net_quantity` | Decimal | | 每份用量 (e.g., 0.05 kg) |

## 3. Operations: Cycle Menu & Logistics (Module: Operations)

This module manages the daily operational flow using a Cycle Menu strategy.

### Table: `ClientCompanyRegion`
(Formerly 'Ward') Represents a physical area for delivery and statistics.

| Field Name | Type | Key | Description |
| :--- | :--- | :--- | :--- |
| `id` | Integer | **PK** | Primary Key |
| `company_id` | Integer | **FK** | Tenant isolation |
| `name` | String(50) | | Region Name (e.g., "ICU", "Staff Canteen") |

### Table: `WeeklyMenu` (The "Pre-recorded" Config)
Defines the standard cycle menu. Users configure this once, and the system reuses it.
*Logic: If Date is Monday -> Load Monday's Menu automatically.*

| Field Name | Type | Key | Description |
| :--- | :--- | :--- | :--- |
| `id` | Integer | **PK** | Primary Key |
| `company_id` | Integer | **FK** | Tenant isolation |
| `diet_category_id` | Integer | **FK** | The Diet Set (e.g., "Standard A") |
| `day_of_week` | Integer | | 1=Monday, 2=Tuesday ... 7=Sunday |
| `meal_time` | String(1) | | Breakfast(B) / Lunch(L) / Dinner(D) |
| `dishes` | M2M | **M2M** | The fixed set of dishes for this slot |

### Table: `DailyCensus` (The Daily Input)
Records the headcount. The user only selects the Diet Type, not the specific dishes.

| Field Name | Type | Key | Description |
| :--- | :--- | :--- | :--- |
| `id` | Integer | **PK** | Primary Key |
| `company_id` | Integer | **FK** | Tenant isolation |
| `date` | Date | | The target date (System calculates Weekday from this) |
| `region_id` | Integer | **FK** | Delivery destination (Links to `ClientCompanyRegion`) |
| `diet_category_id` | Integer | **FK** | Links to `DietCategory` |
| `count` | Integer | | **Headcount**. Used to multiply the WeeklyMenu recipe. |

---

## 4. Operations: Procurement Output (Module: Operations)

The calculation results generated by the system.

### Table: `ProcurementRequest`
The header for a purchase order.

| Field Name | Type | Key | Description |
| :--- | :--- | :--- | :--- |
| `id` | Integer | **PK** | Primary Key |
| `company_id` | Integer | **FK** | Tenant isolation |
| `target_date` | Date | | Date when ingredients are needed |
| `created_at` | DateTime | | Timestamp when the request was created |
| `status` | String(20) | | DRAFT / CONFIRMED |

### Table: `ProcurementItem`
The calculated line items.

| Field Name | Type | Key | Description |
| :--- | :--- | :--- | :--- |
| `id` | Integer | **PK** | Primary Key |
| `request_id` | Integer | **FK** | Links to `ProcurementRequest` |
| `raw_material_id` | Integer | **FK** | Links to `RawMaterial` |
| `total_gross_quantity`| Decimal | | **Calculated Result**. Total amount to buy. |
| `notes` | Text | | Audit trail (e.g., "30kg for Diced, 20kg for Sliced") |

---
